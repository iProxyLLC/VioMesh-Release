# VioMesh — دوران تونل تمام شد

## چرا تونل دیگه جواب نمیده؟

تونل‌های قدیمی (V2Ray، Xray، Shadowsocks، OpenVPN، WireGuard) همه یه مشکل مشترک دارن: **یه اتصال ساده بین دو نقطه.**

فیلترینگ هوشمند الان دقیقا همین الگو رو شناسایی می‌کنه:
- **DPI** (بازرسی عمیق بسته): پروتکل تونل رو تشخیص میده
- **ردیابی اتصال**: یه IP ایران مدام به یه IP خارج وصله — مشکوکه — بلاک
- **محدودیت پهنای باند**: حتی اگه بلاک نشه، سرعت رو میارن پایین
- **بلاک پورت**: پورت‌های معروف تونل رو میبندن
- **قطع دوره‌ای**: هر چند وقت یکبار کل رنج IP رو ریست میکنن

نتیجه؟ هر هفته باید سرور عوض کنی، کانفیگ جدید بزنی، و دوباره همه چی از اول.

## VioMesh: مش، نه تونل

VioMesh یه **شبکه مش** هست، نه تونل. فرقش اینه:

| | تونل | VioMesh |
|---|---|---|
| ساختار | نقطه به نقطه | مش (شبکه‌ای) |
| سرور | یکی، اگه بیفته تموم شد | چندین مسیر خودکار |
| پروتکل | قابل شناسایی | TCP Violation — غیرقابل تشخیص |
| مدیریت | دستی، فنی | خودکار، یه دکمه |
| سرعت | محدود به یه مسیر | بهترین مسیر خودکار |
| قطعی اینترنت | از کار میفته | مسیر جایگزین |

### TCP Violation چیه؟

VioMesh بسته‌هایی میفرسته که **طبق استاندارد TCP نباید وجود داشته باشن** — بسته‌های ACK+PSH بدون SYN. فایروال‌ها و DPI ها اصلا نمیدونن این چیه و ازش رد میشن. این یه تونل معمولی نیست — این یه شکاف بنیادی در نحوه عملکرد TCP هست.

### ویژگی‌ها

- **نصب یه خطی**: کپی-پیست یه دستور و تمام
- **بدون کانفیگ**: همه چیز خودکار تنظیم میشه
- **مش هوشمند**: اگه یه مسیر قطع شه، خودکار از مسیر دیگه میره
- **QUIC over TCP Violation**: سریع‌ترین ترکیب ممکن
- **BBR Congestion Control**: بهینه‌سازی سرعت مخصوص شبکه‌های بد
- **DNS داخلی**: هر سرور اسم داره، فوروارد ساده
- **Port Forward**: هر پورتی رو به هر سروری ببر
- **آپدیت خودکار**: بدون دخالت دستی

## نصب

### ۱. ثبت‌نام

وارد ربات تلگرام شوید:

**[@VioMeshBot](https://t.me/VioMeshBot)**

IP سرورهاتون رو ثبت کنید و کلید نصب دریافت کنید.

### ۲. نصب روی سرور

```bash
curl -sL http://vio.adnsman.net/install.sh | bash -s -- <YOUR_KEY>
```

تمام. VioMesh خودش:
- باینری رو دانلود و نصب میکنه
- کرنل رو بهینه‌سازی میکنه (BBR, TCP buffers, FD limits)
- سرویس systemd میسازه
- به مش وصل میشه

### ۳. مدیریت

```bash
# وضعیت سرویس
systemctl status viomesh

# لاگ‌ها
journalctl -u viomesh -f

# اضافه کردن فوروارد
viomesh forward add tcp 8080 my-server.zone.viomesh 80

# لیست فوروارد‌ها
viomesh forward list

# حذف فوروارد
viomesh forward rm tcp 8080
```

فوروارد‌ها **بدون ریستارت** اعمال میشن — اتصالات فعلی قطع نمیشن.

## چطور کار میکنه؟

```
سرور ایران  ←── TCP Violation ──→  هاب  ←── TCP Violation ──→  سرور خارج
     │                               │                               │
  10.0.x.2                        10.0.x.1                        10.0.x.3
     │                               │                               │
     └─────────── مش شبکه ───────────┴───────────────────────────────┘
```

هر سرور یه IP مش میگیره (مثل `10.0.x.y`) و میتونه مستقیم با بقیه سرورها صحبت کنه. هاب فقط واسطه اولیه‌ست — بعد از اتصال، سرورها مستقیم با هم ارتباط دارن.
